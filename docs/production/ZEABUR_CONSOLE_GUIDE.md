# Zeabur 控制台操作指南

## 📍 如何在Zeabur控制台找到应用URL和状态

### 步骤1：访问Zeabur控制台

1. 打开浏览器，访问：**https://zeabur.com**
2. 登录您的账号
3. 进入您的项目页面

### 步骤2：找到您的服务

在项目页面中：
1. 找到您的服务（service）列表
2. 点击您的服务名称进入详情页

### 步骤3：查看应用URL

#### 方法A：在服务概览页面

1. 在服务详情页的顶部
2. 查找 **"Domain"** 或 **"域名"** 部分
3. 应该看到类似：
   ```
   https://your-service-name-xxx.zeabur.app
   ```
4. **复制这个完整的URL**

#### 方法B：在设置页面

1. 在服务详情页
2. 找到 **"Settings"** 或 **"设置"** 标签
3. 查找 **"Domains"** 或 **"域名"** 部分
4. 查看已配置的域名列表

#### 方法C：在部署日志中

1. 在服务详情页
2. 找到 **"Deployments"** 或 **"部署"** 标签
3. 查看最新的部署记录
4. 可能包含应用URL信息

### 步骤4：检查服务状态

在服务详情页的顶部，查看状态指示器：

- 🟢 **Running** - 服务正常运行
- 🟡 **Building** - 正在构建，请等待
- 🔴 **Failed** - 构建失败
- 🟡 **Restarting** - 正在重启

### 步骤5：查看日志

1. 在服务详情页
2. 找到 **"Logs"** 或 **"日志"** 标签
3. 查看最新的日志信息
4. 检查是否有错误

**正常日志应该显示：**
```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

## 🔍 常见情况处理

### 情况1：找不到域名

**可能原因：**
- 服务还在构建中
- 域名还未分配

**解决：**
1. 等待构建完成
2. 刷新页面查看
3. 如果长时间没有，检查服务是否启动成功

### 情况2：服务状态是 "Building"

**操作：**
1. 等待构建完成（通常3-5分钟）
2. 查看构建日志确认进度
3. 构建完成后会自动分配域名

### 情况3：服务状态是 "Failed"

**操作：**
1. 查看构建/运行日志
2. 查找错误信息
3. 根据错误进行修复
4. 重新部署

### 情况4：有URL但无法访问

**检查：**
1. 确认URL包含 `https://`
2. 确认服务状态是 "Running"
3. 等待2-3分钟让域名生效
4. 清除浏览器缓存后重试

## 📋 检查清单

在Zeabur控制台中确认：

- [ ] 已找到服务详情页
- [ ] 已找到应用URL（格式：`https://xxx.zeabur.app`）
- [ ] 服务状态是 "Running"
- [ ] 日志显示 "Application startup complete"
- [ ] 没有错误信息

## 🎯 找到URL后的操作

### 1. 测试健康检查

使用找到的URL访问：
```
https://your-app-name.zeabur.app/health
```

### 2. 测试API文档

访问：
```
https://your-app-name.zeabur.app/docs
```

### 3. 同步页面Token

访问：
```
https://your-app-name.zeabur.app/admin/deployment/sync-pages
```

## 🆘 如果仍然找不到URL

### 可能的原因：

1. **服务还未部署**
   - 检查是否有未完成的部署
   - 等待部署完成

2. **服务部署失败**
   - 查看构建日志
   - 检查错误信息

3. **权限问题**
   - 确认您有项目访问权限
   - 确认服务已创建
