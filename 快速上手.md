# 🚀 快速上手指南

## 第一步：快速测试（只需1分钟！）

**双击运行** `快速测试.bat` 文件

这个脚本会自动：
- ✅ 检查Python环境
- ✅ 创建配置文件（如果不存在）
- ✅ 安装所需依赖
- ✅ 启动服务

## 第二步：验证服务是否成功

看到这样的输出就成功了：
```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

然后在浏览器打开：**http://localhost:8000**

成功的话会看到：
```json
{
  "message": "多平台客服自动化系统",
  "version": "2.0.0",
  "status": "running"
}
```

## 第三步：查看API文档

浏览器打开：**http://localhost:8000/docs**

你会看到所有可用的API接口，还能直接在网页上测试！

## ⚙️ 配置说明

### 必须配置的项目（在 `.env` 文件中）

1. **Facebook配置** - 必须填写真实值才能接收消息
   - `FACEBOOK_ACCESS_TOKEN` - 你的Facebook页面访问令牌
   - `FACEBOOK_VERIFY_TOKEN` - Webhook验证令牌（自己设置）
   - `FACEBOOK_APP_ID` 和 `FACEBOOK_APP_SECRET`

2. **OpenAI配置** - 必须填写才能用AI回复
   - `OPENAI_API_KEY` - 你的OpenAI API密钥

3. **Telegram配置** - 必须填写才能发送通知
   - `TELEGRAM_BOT_TOKEN` - 你的Telegram机器人令牌  
   - `TELEGRAM_CHAT_ID` - 接收通知的聊天ID

### 可选配置

- **数据库**：默认使用SQLite（测试用），生产环境建议用PostgreSQL
- **Instagram**：如果不用Instagram，可以不配置

## 🐛 常见问题

### 问题1：启动失败
**检查：**
- Python版本是否为3.9+（运行 `python --version` 查看）
- 依赖是否安装成功（重新运行 `快速测试.bat`）

### 问题2：端口被占用
**解决：**
在 `.env` 文件中修改：
```
PORT=8001
```

### 问题3：配置错误
**解决：**
检查 `.env` 文件，确保：
- `FACEBOOK_ACCESS_TOKEN` 等不是 `your_xxx` 格式
- `OPENAI_API_KEY` 以 `sk-` 开头
- `DATABASE_URL` 配置正确

## 📦 部署到生产环境

### 方法1：使用Docker（推荐）
```bash
docker-compose up -d
```

### 方法2：直接部署
1. 修改 `.env` 中的配置
2. 设置 `DEBUG=false`
3. 配置PostgreSQL数据库
4. 运行迁移：`alembic upgrade head`
5. 启动服务：`python -m uvicorn src.main:app --host 0.0.0.0 --port 8000`
